# Contributing to AIRag

Thank you for considering contributing to AIRag! We appreciate your help in improving this project. To ensure a smooth process for everyone involved, please follow these guidelines:

## Contribution Guidelines
- **Fork the repository** and create your own branch for your changes.
- **Make sure your code is modular and well-structured**.
- Always write clear, concise commit messages to describe your changes.

## Code Style
- Follow the existing coding style and conventions used in the project.
- Use meaningful variable and function names.
- Maintain consistent indentation and formatting throughout the codebase.
- Utilize comments and documentation to clarify complex logic.

## Pull Request Process
1. **Open a pull request (PR)** with a descriptive title and details regarding the changes you made.
2. **Reference any related issues** in your PR description to help maintain context.
3. Ensure your PR is against the `main` branch, as this is where the production code resides.
4. **Request review from at least one other contributor** before merging your changes.
5. Once approved, you can merge your PR or ask a maintainer to merge it for you.

## Testing Requirements
- Include tests for any new features or bug fixes in your PR.
- Ensure that all tests pass before submitting the PR.
- Follow the project’s testing framework and methodologies when writing your tests.

## Community Guidelines
- Be respectful and kind in all interactions.
- Provide constructive feedback when reviewing fellow contributors’ code.
- Be open to suggestions and alternative ideas presented by others.
- **Join discussions in an inclusive manner**, respecting all members of the community.

Thank you for contributing to AIRag! We look forward to receiving your contributions!
